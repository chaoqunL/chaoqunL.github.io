FormControl、FormGroup、FormBuilder 的概念和使用。

无论响应式表单还是模板驱动表单都共享了一些底层构造块。

  • FormControl 实例用于追踪单个表单控件的值和验证状态。它和 FormGroup 和 FormArray 是 Angular 表单的三大基本构造块之一。 它扩展了 AbstractControl 类，并实现了关于访问值、验证状态、用户交互和事件的大部分基本功能
  • FormGroup 实例用于追踪一个表单控件集的值和状态。管理一组 AbstractControl 实例的值和有效性状态。该组的属性中包括了它的子控件。组件中的顶级表单就是 FormGroup。
  • FormArray 实例用于追踪一个表单控件数组的值和状态。管理一些 AbstractControl 实例数组的值和有效性状态。
  • ControlValueAccessor 用于在 Angular 的 FormControl 实例和原生 DOM 元素之间创建一个桥梁。
  • FormBuilder 使用用户指定的配置创建 AbstractControl。FormBuilder 服务提供了一些便捷方法来生成表单控件。一个可注入的服务，提供一些用于提供创建控件实例的工厂方法。

ViewChild：属性装饰器，用于配置一个视图查询。 变更检测器会在视图的 DOM 中查找能匹配上该选择器的第一个元素或指令。在调用 NgAfterViewInit 回调函数之前就会设置这些视图查询。
元数据属性（Metadata Properties）:1.selector - 用于查询的指令类型或名字。2.read - 从查询到的元素中读取另一个令牌。
背景：Angular 提供了一种叫做 DOM Query 的技术，主要来源于 @ViewChild 和 @ViewChildren 装饰器（decorators）。
ViewChild 装饰器用于获取模板视图中的元素或直接调用其组件中的方法。它支持 Type 类型或 string 类型的选择器，同时支持设置 read 查询条件，以获取不同类型的实例。比如ElementRef和ViewContainerRef。
ViewChild 装饰器的参数定义是：@ViewChild([reference from template], {read: [reference type]});第二个参数 read 是可选的，因为 Angular 会根据 DOM 元素的类型推断出该引用类型。例如，如果它（#tref）挂载的是类似 span 的简单 html 元素，Angular 返回 ElementRef；如果它挂载的是 template 元素，Angular 返回 TemplateRef。一些引用类型如 ViewContainerRef 就不可以被 Angular 推断出来，所以必须在 read 参数中显式申明（如上面index组件中的使用）。
ViewChild装饰器通常与模板引用变量（template reference variable）一起使用，模板引用变量仅仅是对模板（template）内 DOM 元素命名式引用（a named reference），类似于 html 元素的 id 属性。你可以使用模板引用（template reference）来标记一个 DOM 元素，并在组件/指令类中使用 ViewChild 装饰器查询（query）它，比如：
